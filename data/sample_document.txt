Introduction to Vector Databases and RAG Systems

Vector databases are specialized databases designed to store and query 
high-dimensional vectors called embeddings. Unlike traditional databases 
that match exact values, vector databases find semantically similar content.

What is Endee?

Endee is a lightweight, open-source vector database built for AI applications.
It supports cosine similarity search, metadata filtering, batch operations,
and full CRUD (Create, Read, Update, Delete) functionality.

What is RAG?

RAG stands for Retrieval-Augmented Generation. It enhances large language
models by retrieving relevant documents before generating answers. This
reduces hallucinations and allows LLMs to answer questions about private data.

How This System Works

1. Documents are uploaded and split into chunks
2. Each chunk is converted to an embedding vector
3. Vectors are stored in Endee with metadata
4. User queries are embedded and matched semantically
5. Top matching chunks are passed to Groq LLM as context
6. The LLM generates a grounded, cited answer

Multi-Agent Architecture

The system uses specialized agents:
- Router Agent: Analyzes the query and plans the pipeline
- Researcher Agent: Retrieves relevant chunks from Endee
- Analyst Agent: Identifies patterns and insights
- Synthesizer Agent: Produces the final coherent answer

Tech Stack

Vector Database: Endee
Language Model: Groq (llama-3.3-70b-versatile)  
Embeddings: sentence-transformers (all-MiniLM-L6-v2)
Web Framework: Flask
Document Processing: PyPDF2, python-docx
